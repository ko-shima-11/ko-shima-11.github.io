モノドコ - 倉庫ロケーション管理

商品とユニットの保管場所をQRと検索で即座に把握できるWebアプリ

　

システム概要

モノドコは倉庫内の商品ロケーションを商品コード・ロット単位で管理し、検索・入庫・出庫・移動までを一つのUIで完結できるWebアプリです。
スマートフォンとPCの両方から使える構成で、ホーム・検索・読み取り・管理・DB管理をシンプルな導線にまとめています。
商品・ユニットのQRコードを併用し、正しい保管場所を即座に特定できるようにしました。
生成したQRを貼り付けて現場の棚を識別し、最新の在庫状況を途切れなく共有できます。
すべてのデータはSQLiteに保存され、管理者はDB管理画面から直接編集やリセットも行えます。

開発背景

倉庫で「どの棚に何があるか」を把握できず、現場の探索や問い合わせに時間がかかる課題がありました。
口頭や紙での管理では記録漏れが起こりやすく、ロット移動や出庫時に最新の保管場所を即座に確認できませんでした。
モノドコはQRコードと絞り込み検索を組み合わせ、スマホでも操作しやすい導線で入出庫を記録できるように設計しています。
導入時のハードルを下げるため、データベースはSQLiteのみで完結し、閉域ネットワーク内での利用を前提にしました。
現場の担当者が迷わず扱えるUIと、履歴を残しながら正確なロケーション管理ができる仕組みを提供します。

主要機能

在庫検索

商品コード・商品名・ロット番号・ユニット番号・在庫状況で絞り込んで検索できます。
絞り込み条件はいつでもリセットでき、一覧から必要な商品をすぐに特定可能です。
検索結果から出庫・移動をそのまま開始できます。

QR読み取りと入庫・移動

カメラで商品QRを読み取ってロット番号とユニットを登録し、入庫を記録します。
ユニットQRを読み取れば、棚ごとの在庫内容を確認し、そのまま移動操作へ進めます。
QRと手入力の両方を使えるため、現場での運用に柔軟に対応できます。

商品・ユニット登録／QR生成

管理画面から商品とユニットを新規登録・更新・削除できます。
登録時にQRコードを自動生成し、印刷して棚や商品に貼り付け可能です。
既存データからのQR再発行にも対応しています。

データベース管理・履歴

DB管理画面で全商品のマスタ、ユニット、在庫データを一覧・直接編集できます。
不要データの削除や在庫テーブルのリセット、最新100件の履歴参照が行えます。
現場で起きた操作を追跡し、誤操作時の修正も即時に反映できます。

工夫した点

現場導線を最短に：
・ホーム→検索/読み取り→管理→DB管理の流れを最短ステップに整理しました。
・検索条件リセットや一覧からの直接操作で、ロケーション特定までの手数を減らしています。
・スマホ表示を前提としたボタン配置で、片手でも扱いやすいUIにしています。

QRファースト + 手入力の併用：
・商品QRとユニットQRを組み合わせて、入庫・出庫・移動を正確に記録します。
・読み取りが難しい環境でも、手入力で同じ処理を進められる冗長構成です。

シンプルなデータモデル：
・商品マスタ、ユニットマスタ、在庫テーブル、履歴テーブルを分離し、ロケーション管理に特化しました。
・在庫ステータスはactive/shippedで管理し、0個時の出庫記録にも対応しています。

閉域・ローカル運用に最適化：
・SQLite単体で動作し、追加ミドルウェアなしで構築できます。
・Next.js API Routes + better-sqlite3の同期処理で、安定したレスポンスを確保しています。

技術選定理由

【フロントエンド】
Next.jsのApp RouterとReact 18で、検索や読み取り画面間の遷移を軽快にしつつ同一コードベースで管理しています。
TypeScriptでAPIリクエストやフォームの型を明確化し、入力ミスやデータ不整合を抑制します。
CSS Modulesによるスタイル分離で、画面単位の調整を素早く行えるようにしています。

【バックエンド】
Next.js API Routesを使い、在庫検索・入庫・出庫・移動・マスタ登録を同じプロジェクト内で完結させました。
better-sqlite3を同期処理で用いてレスポンスの安定性を優先し、セットアップの簡便さを担保しています。
qrcodeでラベル用QRを生成し、jsQRで読み取りできるため、登録から実運用までのQRフローを一貫させています。

【データベース】
SQLite単体で動作するため追加サーバーが不要で、`demo.db` を生成するだけで始められます。
単一ファイルのためバックアップやリセットが容易で、DB管理画面とも整合性を取りやすい構成です。

【認証】
現時点では閉域ネットワーク・単一管理者での運用を想定し、認証は未実装としています。
複数ユーザー運用が必要になった際に、権限管理と併せて拡張できるようにしています。

【環境構築】
Node.js環境があれば `npm install` と `npm start` で起動でき、開発時は `npm run dev` のホットリロードで開発効率を高めています。
追加の外部サービスに依存せず、ローカルと閉域の双方で同じ手順で動かせます。

【CI/CD】
将来的にGitHub Actionsでlintや簡易テストを自動化し、デプロイ作業を安定化させる計画です。
単一リポジトリに集約しているため、パイプラインの適用範囲も明確です。
